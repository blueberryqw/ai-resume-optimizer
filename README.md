# AI 简历优化助手

一个面向大学生、研究生和转岗求职者的 JD-简历匹配分析工具。用户输入目标岗位 JD、个人简历和岗位类型后，系统会拆解岗位要求、计算简历匹配度，并输出关键词命中、六维诊断、改写对比和可下载的分析报告。

第二版继续使用本地规则匹配与模板生成，不调用 OpenAI API 或其他 AI 服务，不需要 API Key，也不会产生模型调用费用。

## 项目背景

求职者常见的问题不是“没有经历”，而是简历中的经历没有按照目标岗位的语言进行表达：

- 岗位关键词覆盖不足，难以通过招聘方的快速筛选。
- 经历偏职责和过程，缺少结果、数据与个人贡献。
- 同一份简历投递多个方向，缺少针对性的能力证明。
- 对 AI、产品、运营等能力的描述比较抽象，面试时也难以形成清晰故事。

本项目将 JD 分析、简历诊断、经历改写和求职沟通整合为一个轻量产品流程，帮助用户快速完成第一轮简历优化。

## 用户痛点

- **不知道岗位真正看重什么**：JD 信息较长，求职者难以快速提取核心能力和高优先级关键词。
- **不知道自己的差距在哪里**：只能凭感觉修改简历，缺少统一、可解释的诊断维度。
- **经历表达偏职责**：常用“负责、参与、协助”等表达，缺少目标、动作、结果和个人贡献。
- **一份简历投递多个岗位**：关键词和经历排序没有针对不同岗位进行调整。
- **修改建议不够落地**：常见建议只强调“增加数据、突出亮点”，但没有给出可直接参考的表达。
- **求职材料分散**：简历改写、招聘平台招呼语和面试项目亮点需要分别准备。

## 目标用户

- 正在准备实习或校招的大学生、研究生
- 希望从传统岗位转向产品、运营或 AI 方向的求职者
- 有实践经历，但不擅长提炼成果和岗位匹配点的用户
- 需要快速针对不同 JD 调整简历表达的求职者

## 功能模块

1. **JD 核心关键词提取**：识别数据分析、用户增长、AIGC、竞品分析等岗位重点。
2. **岗位能力归纳**：将关键词映射为数据分析、用户理解、产品思维等能力要求。
3. **简历匹配度评分**：从关键词覆盖、能力信号、数据表达和完整度四个维度给出 0-100 分。
4. **决策摘要**：集中展示综合评价、优先优化建议与简历风险提示。
5. **岗位关键词命中表**：逐项展示关键词是否命中、重要程度及具体修改建议。
6. **六维问题诊断**：分析数据化表达、关键词覆盖、结果意识、AI 工具使用、项目经验和产品/运营思维。
7. **改写前后对比**：并列展示原始表达与优化表达，不编造夸张数据。
8. **分析报告导出**：生成 Markdown 格式报告，支持页面预览与下载。
9. **求职辅助**：生成 Boss 直聘招呼语、面试项目亮点和可用于简历的项目描述。

## 产品流程

```text
输入岗位 JD 和简历
        ↓
选择目标岗位类型
        ↓
提取 JD 关键词与能力要求
        ↓
匹配简历证据并计算分数
        ↓
生成关键词命中表与六维诊断
        ↓
输出优先建议、风险提示和改写对比
        ↓
导出报告并生成求职展示材料
```

## 技术实现

- **页面框架**：Python + Streamlit
- **关键词提取**：岗位关键词与同义表达规则库
- **能力归纳**：关键词到岗位能力的映射规则
- **匹配评分**：
  - JD 关键词覆盖最高 55 分
  - 岗位通用能力词最高 20 分
  - 数据化表达最高 15 分
  - 简历完整度最高 10 分
  - 对内容过短、缺少数字、关键词覆盖过低进行扣分或限分
- **文案生成**：基于简历原句、目标岗位和命中关键词的动态模板
- **数据存储**：无数据库，分析在本地会话中完成

## 运行方法

建议使用 Python 3.9 或更高版本。

```bash
py -m pip install -r requirements.txt
py -m streamlit run app.py
```

启动后，终端会显示本地访问地址，通常为：

```text
http://localhost:8501
```

可以从 `sample_data.md` 复制示例 JD 和简历进行测试。

## 项目亮点

- 将复杂的求职问题拆解为“岗位理解—简历诊断—表达优化—求职沟通”的完整产品链路。
- 使用可解释的评分模型，用户可以查看得分构成和扣分原因。
- 增加关键词命中表和六维诊断，将抽象的“匹配度”转化为具体修改优先级。
- 采用改写前后对比，让用户能够理解修改逻辑，而不只是接受一段生成结果。
- 支持导出 Markdown 分析报告，形成可保存、可复查的完整分析产物。
- 在不接入大模型 API 的情况下，通过规则库、能力映射与动态模板完成可运行 MVP。
- 对缺少真实数据的经历采用克制表达，避免生成虚假或夸张成果。
- 页面采用模块化结果展示，适合面试演示和作品集截图。
- 规则、评分与文案生成逻辑均集中在本地，便于后续替换为真实 AI 能力。

3. 支持上传 PDF、Word 简历并自动提取文本。
4. 增加简历版本管理和修改前后对比。

## 面试讲解话术

> 这个项目来自我观察到的一个求职问题：很多用户并不是没有经历，而是不知道如何根据具体 JD 重组简历。我的目标不是单纯做一个文案生成器，而是先帮助用户理解岗位、定位差距，再给出可执行的修改方案。
>
> 在产品流程上，我将功能拆成输入、岗位解析、匹配评分、问题诊断、表达优化和报告导出六个阶段。评分模型由关键词覆盖、能力信号、数据化表达和简历完整度组成，同时增加关键词命中表和六维诊断，让结果可解释。
>
> 第一阶段没有直接接入大模型，而是使用 Python、Streamlit、岗位规则库和动态模板完成本地 MVP。这样可以先验证用户是否需要这套分析流程，也能降低成本并保护简历信息。第二版重点优化了决策摘要、风险提示、改写前后对比和 Markdown 报告导出，使产品更接近可展示、可使用的完整 Demo。
>
> 如果继续迭代，我会优先接入真实语义分析、PDF 简历解析和历史版本管理，并通过用户反馈校准评分权重。
